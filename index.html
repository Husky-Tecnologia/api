Autenticação

Todas as requisições à API da Husky devem ser autenticadas por meio de um token.

Autenticação como Operação:

  • Para autenticar como Operação, deve ser passado via query parameter o token da Operação e o parâmetro customer deve ser false.

Autenticação como Embarcador:

  • Para autenticar como Embarcador, deve ser passado via query parameter o token do Embarcador e o parâmetro customer deve ser true.

Autenticação como Embarcador (v2):

  • Para autenticar como Embarcador (v2), deve ser passado via header o token do Embarcador, não precisa passar o parâmetro customer.

Webhooks

Eventos de notificação Webhook

Trabalhamos com webhooks para notificações de eventos do sistema Husky. Para receber as notificações, é necessário cadastrar a URL de destino e o tipo de evento que será notificado.

Eventos disponíveis e o momento em que o evento será notificado:

ORDER_CREATED: Quando uma nova entrega é criada
ORDER_UPDATED: Quando a entrega é atualizada
ORDER_STATUS_UPDATED: Quando o status da entrega é atualizado (ex: coletando, entregando, etc)
ORDER_DRIVER_UPDATED: Quando o entregador da entrega é atualizado
FLEET_QUEUE: Quando o entregador sai da fila do embarcador
DRIVER_CREATED: Quando um novo entregador é criado
DRIVER_UPDATED: Quando os dados do entregador são atualizados
DRIVER_STATUS_UPDATED: Quando o status do entregador é atualizado (ex: disponível, ocupado, etc)
CLIENT_CREATED: Quando um novo cliente é criado
CLIENT_UPDATED: Quando os dados do cliente são atualizados
CLIENT_STATUS_UPDATED: Quando o status do cliente é atualizado (ex: bloqueado, etc)
all: Todos os eventos serão notificados

Consulta de Status da entrega

Este endpoint recebe via POST um campo tracking_codes com um array de tracking de entregas e retorna o status delas.

Por quanto tempo que as entregas estão disponíveis para consulta?
Entregas em aberto sempre estão disponiveis para consulta. Entregas finalizadas estão disponíveis em um range de 7 dias (3 dias antes da data atual e 3 dias após a data atual) da data de agendamento da entrega. Por exemplo: uma entrega finalizada e que havia sido agendada para o dia 13/02/2025 só poderá ser consultada no endpoint até o dia 16/02/2025.
O payload de resposta retorna o motivo de Não Finalizado na sua forma de código. O código segue os Motivos de Status Não Finalizado que podem ser obtidos no endpoint /unfinishedStatusReasons que hoje está publico na documentação.
